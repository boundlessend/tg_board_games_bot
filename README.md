НА-СТОЛ-КИ by senya

Telegram-бот с помощниками для настольных игр. Помощники: «Опасные слова» (слова, проклятья, боссы) и словесные игры «Крокодил», «Алиас», «Кто я?». Есть дискуссионная игра «Бункер» с автоматическим ведением партии в групповом чате. Контент выдаётся без повторов для каждого Telegram ID, история хранится в SQLite. Есть инлайн-режим и админ-меню со статистикой и добавлением контента.

## Установка

```bash
python3.12 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

## Настройка

```bash
cp .env.example .env
```

В `.env` укажите токен и (опционально) Telegram ID администраторов через запятую:

```env
BOT_TOKEN=your_real_telegram_bot_token
ADMIN_IDS=11111111,22222222
```

Без `ADMIN_IDS` админ-меню недоступно никому.

## Запуск

```bash
python bot.py
```

При первом запуске рядом с кодом создаётся `bot.sqlite3`.

### Docker Compose (рекомендуется)

```bash
cp .env.example .env    # укажите BOT_TOKEN (и опционально ADMIN_IDS)
docker compose up -d    # сборка образа и запуск в фоне
docker compose logs -f  # логи
docker compose down     # остановить
```

Compose собирает образ, поднимает бота на long polling и хранит базу в томе `db` (`DATABASE_PATH=/db/bot.sqlite3` зашит в образ), поэтому история выдач переживает перезапуски. Токен передаётся через `env_file`, в образ `.env` не попадает.

### Docker вручную

```bash
docker build -t tg-board-games .
docker run --rm --env-file .env -v tg-board-games-db:/db tg-board-games
```

Образ работает от непривилегированного пользователя. `--rm` подходит для запуска по требованию (под отдельную игру); для постоянного аптайма используйте `-d --restart unless-stopped`.

## Данные

- `data/words.json` — 800 обычных и 800 фэнтези-слов.
- `data/curses.json` — 200 проклятий.
- `data/bosses.json` — 200 боссов.
- `data/crocodile.json`, `data/alias.json`, `data/whoami.json` — пулы слов для словесных игр «Крокодил» (218), «Алиас» (222), «Кто я?» (185).
- `data/bunker.json` — пулы карт игры «Бункер» (катастрофы, суперсилы, фобии, характер, хобби, багаж, факты, особые условия, угрозы, карты бункера).

Пулы можно расширять из админки (контент хранится в SQLite и добавляется к файловым словам при выдаче).

В сообщениях показывается счётчик выдачи относительно размера пула (база: 1600 слов, по 200 проклятий и боссов; пул растёт за счёт добавленного админом контента). В «Опасных словах» слова после исчерпания пула заканчиваются, проклятья и боссы начинают новый круг. В «Крокодиле» и «Алиасе» слова после исчерпания идут на новый круг.

Меню работает на inline-кнопках (сообщение редактируется на месте). Команда `/help` показывает справку.

Два режима по типу чата:

- В личке с ботом доступна игра `Кто я?` - тяни персонажа кнопкой.
- В беседе (добавьте бота в группу) доступны `Крокодил`, `Алиас`, `Опасные слова` и `Бункер`.

Инлайн-режим: наберите `@имя_бота` в любом чате, чтобы получить случайные слова (без учёта истории).

В личном меню есть `Настройки`: переключатель авто-цикла словесных игр (при «выкл» после исчерпания слов нужно нажать «Новая игра»).

Избранное: `/fav` сохраняет последнее выданное слово, `/favorites` показывает список, `/favclear` очищает.

## Командные словесные игры (беседа)

`Крокодил` и `Алиас` в беседе идут по командам:

1. Создатель открывает игру из меню, в лобби выбирает число команд (2-4) и время хода (30/60/90 с), игроки вступают в команды кнопками, создатель жмёт «Начать».
2. Игрок команды, чей ход, жмёт «Слово в ЛС» - бот присылает слово ему в личку (нужно заранее открыть ЛС с ботом через `/start`, иначе слово не дойдёт) и запускает таймер хода.
3. «Угадали +1» начисляет очко и выдаёт следующее слово, «Пропустить» - без очка, «Передать ход» - другой команде, «Завершить» - итоговое табло. По истечении времени хода бот сам передаёт ход следующей команде.

Команда `/play` открывает тот же выбор игр.

## Опасные слова (беседа)

В беседе `Опасные слова` - игра на 2 команды, бот работает крупье (счёт и запретные слова ведутся «вживую» на своём поле):

1. Создатель открывает игру из меню. Каждый ход одна команда загадывает, другая объясняет; роли чередуются кнопкой «Передать ход».
2. Игрок объясняющей команды жмёт «Я объясняю». Игрок загадывающей команды жмёт «Тянуть слово» - слово приходит только ему в личку, команда пишет к нему запретные слова. Когда готовы, «Отправить слово» отправляет это же слово объясняющему из другой команды в личку. В общий чат слово не попадает.
3. Ведущий (кто открыл игру) кнопкой «Проклятие» тянет случайное проклятие с выбором «Принять / Реролл»: реролл заменяет его новым, принятое остаётся в чате (так проклятия не копятся). Кнопка «Босс» так же с «Принять / Реролл» выбирает единственного босса партии (финальный ход), «Завершить» - конец.

Состояние сессий живёт в памяти процесса на время игры (теряется при перезапуске бота).

## Бункер

Дискуссионная игра на выживание для группового чата. Добавьте бота в беседу и отправьте `/bunker`:

1. Игроки жмут «Вступить в бункер» (нужно от 4 до 16). Карты персонажа приходят каждому в личку, поэтому игрокам нужно заранее открыть ЛС с ботом (`/start`).
2. Создатель жмёт «Начать»: бот раздаёт карты, открывает катастрофу и план партии (мест в бункере и сколько изгнать по числу игроков).
3. Каждый раунд (всего 5): открывается пара «Бункер + Угроза», игроки кнопкой «Открыть мою карту» раскрывают по одной карте (порядок фиксированный: суперсила, фобия, характер, хобби, багаж; факт раскрывается в финале).
4. Голосования за изгнание идут по таблице раундов из правил. Голоса принимаются тайно, при ничьей - переголосование среди равных, затем случайный выбор.
5. После 5 раундов оставшиеся попадают в бункер - в базовом режиме это победители.

В лобби кнопкой переключается режим: «базовый» (победители - все попавшие в бункер) или «история выживания» (развёрнутый финал, см. ниже).

### История выживания

Если в лобби выбран этот режим, после 5 раундов разыгрывается финал:

- Группа в бункере тянет одну из открытых за игру угроз, изгнанные - две случайные.
- По каждой угрозе и затем по катастрофе идёт общее голосование «Справились / Не справились» (голосуют все, включая изгнанных): «справились» - если хотя бы половина считает, что у группы хватает полезных карт.
- При провале разыгрывается случайная потеря: гибнет один член группы или (с шансом «карты угрозы») вся группа; провал катастрофы губит всех выживших.
- В итоге бот объявляет, кто пережил историю.

Особые условия раздаются в личке вместе с картами и разыгрываются голосом. Состояние партии живёт в памяти процесса (теряется при перезапуске бота).

### Режим «отдельно» (без беседы)

Если игроки рядом и общей беседы нет, играют через код-приглашение в личке с ботом:

1. Создатель пишет боту `/bunker` в личке - бот выдаёт код игры.
2. Остальные открывают ЛС с ботом и вводят `/joinbunker КОД` (нужно 4-16 игроков).
3. Создатель жмёт «Начать»: каждому в личку приходит его секретный набор карт персонажа, общая катастрофа, план партии и пары «Бункер + Угроза» по раундам.

Дальше партия идёт «вживую»: игроки сами раскрывают карты по одной каждый раунд и голосуют за изгнание. Бот в этом режиме выступает крупье (раздаёт карты и общий стол), без автоведения раундов. Развязка «История выживания» в режиме «отдельно» тоже проводится игроками вручную - её автоведение есть только в режиме беседы.

## Админ-меню

Добавьте свой Telegram ID в `ADMIN_IDS` в `.env` и отправьте боту команду `/admin` в личном чате. Доступ к админ-меню есть только у ID из списка. Бот покажет сводку (число пользователей, суммы выдач, выдачи и активные пользователи за 7 дней, топ-10 слов). Кнопки: `Полный отчёт` - подробная статистика по каждому Telegram ID (длинный отчёт делится на несколько сообщений), `Выгрузить CSV` - статистика файлом, `Активность` - выдачи по дням за 14 дней, `Закрыть админку` - выход.

Команды добавления контента (только для админов, в SQLite):

- `/addword <игра> <слово>` - игры: `dangerous_words`, `crocodile`, `alias`, `whoami`.
- `/addcurse <название> | <описание>`
- `/addboss <имя> | <описание>`
- `/listcontent` - показать добавленный контент.
- `/delword <игра> <слово>`, `/delcurse <id>`, `/delboss <id>` - удалить добавленное.
- `/importwords <игра>` - подпись к файлу (JSON-массив или слова по строкам/через запятую) для массового добавления слов.
- `/backup` - получить файл базы.
- `/restore` - прислать файл `.sqlite3` с подписью `/restore` (база будет заменена).

## Добавление игр

Простая словесная игра (как «Крокодил»/«Алиас»):

1. Положите файл `data/<game>.json` вида `{"words": [...]}`.
2. Добавьте запись в `_WORD_GAME_FILES` в `services/random_generator.py`.

Кнопка в главном меню, меню игры, выдача без повторов и авто-цикл подключаются автоматически через `handlers/word_games.py`.

Сложная игра со своей механикой (как «Опасные слова»): создайте роутер в `handlers/` и подключите его в `bot.py`; выдачу без повторов можно переиспользовать из `services/random_generator.py`.

# PRD: НА-СТОЛ-КИ (tg_board_games_bot) - V1

Документ описывает функционал бота. Раздел «V1» фиксирует исходный публичный релиз. Раздел «V2» (в конце файла) описывает развитие; изменения относительно V1 отслеживаются там со статусами.

## 1. Обзор

Telegram-бот с набором помощников для настольных игр. В V1 реализован один помощник: «Опасные слова». Бот выдаёт игровой контент (слова, проклятья, боссов) случайно и без повторов для каждого Telegram ID, храня историю выдач в SQLite. Архитектура заложена под добавление новых игр.

## 2. Цель продукта

Заменить бумажные карточки и ручной выбор контента во время партии: ведущий и игроки получают уникальный неповторяющийся контент по кнопке, а организатор видит статистику использования.

## 3. Пользователи и роли

- Игрок (`Я игрок`): получает слова и сбрасывает их историю.
- Ведущий (`Я ведуший`): получает слова, проклятья, боссов и сбрасывает каждый список отдельно.
- Администратор: входит по секретной фразе в личном чате, видит сводную статистику по всем пользователям. Права администратора хранятся в памяти процесса и сбрасываются при перезапуске бота.

## 4. Функциональные требования (реализовано)

### 4.1 Навигация
- `/start` и кнопка `Назад в главное меню` показывают главное меню с выбором игры.
- В главном меню одна игра: `Опасные слова`.
- Внутри игры выбор роли: `Я ведуший` / `Я игрок`, кнопка `Назад к выбору роли`.
- Любой нераспознанный текст возвращает подсказку «Выберите действие с помощью кнопок меню».
- Интерфейс на reply-клавиатурах (`ReplyKeyboardMarkup`).

### 4.2 Выдача контента без повторов
- Каждая выдача случайна из ещё не выданных пользователю элементов и сразу сохраняется в историю по `telegram_id`.
- Уникальность гарантируется на уровне БД (UniqueConstraint), повторная попытка сохранить дубль перезапрашивает доступный пул.
- В сообщении показывается счётчик выдачи в формате `(n/лимит)`.

### 4.3 Слова
- Кнопка `Создать новое слово` выдаёт новое слово. Доступно в обеих ролях.
- Лимит 1000 (500 обычных + 500 фэнтези). После исчерпания пул не обнуляется автоматически: бот отвечает «Список кончился, напомни Сене его пополнить».
- Кнопка `Сбросить слова` очищает историю слов пользователя, счётчик возвращается к `0/1000`.

### 4.4 Проклятья и боссы (только роль ведущего)
- Кнопки `Создать новое проклятье` / `Создать нового босса` выдают проклятье/босса с заголовком и описанием.
- Лимит 100 на каждый список. После исчерпания пул автоматически обнуляется (новый круг) и выдаётся следующий элемент; пользователь получает предупреждение о новом круге.
- Кнопки `Сбросить проклятья` / `Сбросить боссов` очищают соответствующую историю вручную.

### 4.5 Админ-меню
- Активация: ввод секретной фразы (`abobus`) в личном чате. Любой пользователь, знающий фразу, получает права администратора.
- После входа бот сразу присылает сводную статистику по всем Telegram ID.
- Кнопка `Вся стата` повторяет отчёт, `Закрыть админку` выходит из режима.
- Отчёт содержит по каждому пользователю: счётчики и списки выданных слов, проклятий (id + название), боссов (id + имя).
- Отчёт длиннее лимита Telegram (3500 символов) автоматически делится на несколько сообщений.
- Команды админки доступны только в личном чате.

## 5. Нефункциональные требования и реализация

- Язык/рантайм: Python 3.12+.
- Фреймворк бота: aiogram 3.x, long polling.
- Хранилище: SQLite через SQLAlchemy 2.x (async) + aiosqlite. Файл `bot.sqlite3` создаётся рядом с кодом при первом запуске.
- Конфигурация: `BOT_TOKEN` из `.env` (python-dotenv). Отсутствие токена приводит к явной ошибке `ConfigError`.
- Обработка ошибок: специфичные исключения (`ConfigError`, `DatabaseError`, `ContentError`/`EmptyPoolError`/`DataFileError`, `DuplicateHistoryItemError`); ошибки БД логируются структурированно и не роняют бота, пользователь получает понятное сообщение.
- Валидация контента при загрузке: проверка структуры JSON, непустых строк, уникальности id и слов.

## 6. Архитектура

- `bot.py` - точка входа: загрузка конфигурации и контента, инициализация хранилища, регистрация роутеров, запуск polling.
- `config.py` - чтение и валидация конфигурации (`BotConfig`).
- `constants.py` - тексты кнопок, лимиты, ключи истории, имена таблиц, секретная фраза.
- `keyboards.py` - сборка reply-клавиатур.
- `database.py` - `SQLiteHistoryStorage`: подключение к SQLite и операции с историей.
- `services/random_generator.py` - загрузка/валидация контента и чистая логика выбора без повторов (`select_unique_item`).
- `handlers/` - `start.py`, `dangerous_words.py`, `admin.py`.
- `data/` - `words.json`, `curses.json`, `bosses.json`.
- `exceptions.py` - общие исключения.

## 7. Модель данных

Три таблицы, по одной на тип контента, ключ изоляции - `telegram_id`:

- `user_words(id, telegram_id, word)`, UniqueConstraint(`telegram_id`, `word`).
- `user_curses(id, telegram_id, curse_id)`, UniqueConstraint(`telegram_id`, `curse_id`).
- `user_bosses(id, telegram_id, boss_id)`, UniqueConstraint(`telegram_id`, `boss_id`).

История - это и есть статистика: отдельной таблицы метрик нет.

## 8. Контент

- `words.json`: 500 обычных + 500 фэнтези-слов (приводятся к нижнему регистру, дубли запрещены).
- `curses.json`: 100 проклятий (`id`, `title`, `description`).
- `bosses.json`: 100 боссов (`id`, `name`, `description`).

## 9. Известные ограничения и нюансы V1

- Права администратора выдаются по знанию фразы, а не по `telegram_id`; фраза захардкожена в `constants.py`. Это не настоящая авторизация.
- Состояние «вошёл в админку» хранится в памяти процесса и теряется при рестарте.
- Названия кнопок выдачи начинаются со слова «Создать», хотя фактически контент не создаётся пользователем, а выдаётся из готового пула.
- Слова в отличие от проклятий и боссов не обнуляются автоматически после исчерпания.
- Деплой только через polling; нет Dockerfile, webhook, миграций схемы (схема создаётся через `create_all`).
- Опечатка в тексте кнопки роли: `Я ведуший`.

## 10. Вне scope V1

- Несколько игр одновременно (заложена только архитектура для расширения).
- Групповые/совместные сессии, инлайн-режим, таймеры, подсчёт очков.
- Пользовательский контент (свои наборы слов).
- Аналитика помимо сырого дампа истории.
